# Humor Generation Transformer

## Overview
This repository contains a custom-built, autoregressive Large Language Model (LLM) developed from scratch using PyTorch. The primary objective of this project is to generate context-aware humor based on short text prompts. The project demonstrates a fundamental understanding of Natural Language Processing (NLP) by implementing custom Causal Self-Attention mechanisms and Transformer blocks, alongside a structured two-phase training pipeline.

## Model Architecture
The core model, `MiniTransformerLM`, is a decoder-only transformer built without relying on high-level pre-packaged transformer classes. 

Key architectural specifications include:
* **Embedding Dimension**: 256
* **Attention Heads**: 4
* **Transformer Layers**: 4
* **Feed-Forward Hidden Dimension**: 512
* **Max Sequence Length**: Configurable (typically 64 to 128 tokens)
* **Components**: Custom `CausalSelfAttention`, `TransformerBlock` (incorporating Layer Normalization, GELU activations, and Dropout), and a linear language modeling head.

## Data Pipeline and Tokenization
* **Custom Tokenizer**: A Byte-Level BPE (Byte-Pair Encoding) Tokenizer was trained from scratch on the target corpora using the Hugging Face `tokenizers` library.
* **Special Tokens**: Engineered prompt structures utilizing specific tokens (`<s>`, `<pad>`, `</s>`, `<mask>`).
* **Data Loaders**: Developed custom PyTorch `Dataset` classes (`GeneralTextDataset` and `HumorDataset`) to handle text chunking, dynamic padding, and mask generation for accurate Cross-Entropy loss calculation (ignoring padding tokens).

## Training Methodology
The model employs a two-stage training strategy, optimized via AdamW and a Cosine Annealing Learning Rate Scheduler.

1. **Phase 1: Pre-training (Domain Adaptation)**
   * The model was trained on a large corpus of general English text, including Project Gutenberg literature (e.g., *War and Peace*) and stand-up comedy transcripts.
   * **Objective**: To build a foundational understanding of English semantics, grammar, and sentence structure.

2. **Phase 2: Fine-tuning (Task Specific)**
   * The pre-trained model was fine-tuned with a reduced learning rate on a proprietary, tab-separated dataset (`my_dataset2.tsv`) consisting of input prompts and corresponding joke punchlines.
   * **Objective**: To align the model's text generation capabilities with the specific structure and cadence of humor.

## Generation Strategy
Text generation is handled autoregressively. The pipeline formats the prompt exactly as seen during training (`<s>input: {text}\njoke: [`) and utilizes temperature-scaled multinomial sampling to ensure diverse and creative outputs, terminating generation upon predicting the closing bracket or an EOS token.
